# Claude Code自動化で副業の作業時間を週5時間に減らした方法

## はじめに

「副業をやりたいけど、時間が足りない」
それは時間管理の問題じゃない。仕組みの問題だ。

通信業界の会社員として平日8時間働きながら、Claude Codeで副業をしている。
最初の3ヶ月は週20時間近く副業に使っていた。体が持たなかった。

この記事では、そこから週5時間に削った方法を全部書く。
読み終えたら今日から1つ仕組みを作れる状態になる。

---

## 自動化前の状態（週何時間かかっていたか）

副業を始めた最初の3ヶ月のスケジュールはこうだった。

| 作業 | 週あたりの時間 |
|---|---|
| コンテンツのネタ探し | 3〜4時間 |
| 記事構成を考える | 2〜3時間 |
| 執筆 | 6〜8時間 |
| SNS投稿文の作成 | 2〜3時間 |
| 画像作成・投稿作業 | 3〜4時間 |
| **合計** | **16〜22時間** |

会社員をしながらこれをやると、睡眠が削れる。週3日は23時を超えていた。

### 問題の本質は「毎回ゼロからやっていた」こと

ネタ探しも、構成も、文章の出だしも。毎回同じ思考プロセスを繰り返していた。
これは仕組みで解決できる。

---

## 自動化した3つの仕組み

### 仕組み1：ネタ収集の完全自動化（週3〜4時間 → 0時間）

Claude CodeのRoutinesで「毎朝5時にX・Web・Noteを調査してCSVにネタを保存する」仕組みを作った。

起きたらネタが10件揃っている。私の作業はゼロ。

収集するのは以下の3ソース。

- **X（リアルな声）**: 副業・AI・Claude Codeに関する投稿からリアルな悩みを抽出
- **Webニュース**: 直近1週間のAI最新情報・副業市場の動向
- **Note**: 競合記事・読者反応が高いテーマ

各ネタには自動でスコアが付く。「Note記事向き」か「X投稿向き」かも判定される。
翌朝には「今日どのネタで書くか」を選ぶだけの状態になっている。

### 仕組み2：記事構成の半自動化（週2〜3時間 → 30分）

収集済みのネタを元に、Claude Codeが企画案を3つ自動生成する。

各案には以下が含まれる。

- タイトル案（3パターン）
- ターゲット読者の具体像
- H2見出し構成（3〜5本）
- 冒頭フック文
- 推定文字数

私の作業は「どの案を採用するか」を決める30分だけ。
構成の善し悪しを判断する時間は省けないが、ゼロから考える時間はゼロになった。

### 仕組み3：執筆の半自動化（週6〜8時間 → 1〜2時間）

承認した企画をClaude Codeに渡すと、初稿が生成される。

ポイントはCLAUDE.mdに自分の文体ルールを書いておくこと。

```
- 1文60字以内
- 結論ファースト
- 敬語なし
- 具体例・数字を必ず含める
- 「会社員 × 副業 × AI」の文脈からズレない
```

これを書いておくだけで、毎回指示しなくてもスタイルが統一される。
私の作業は初稿のチェックと体験談の肉付けだけ。1〜2時間に収まる。

## 実際の1週間のスケジュール（自動化後）

| タイミング | 作業 | 所要時間 |
|---|---|---|
| 毎朝7時 | 企画案を確認・承認 | 5分 |
| 平日夜 | 記事チェック＋体験談を追加 | 20分 |
| 土曜 | Note投稿＋X投稿まとめ作業 | 1〜2時間 |
| 日曜 | 過去記事の見直し | 30分 |
| **週合計** | | **約5時間** |

平日は1日20分。週末に少しまとめてやる。これなら続く。

会社から帰って夜20分だけ副業に使うイメージ。
「今日は疲れたから休もう」という罪悪感もなくなった。仕組みが回っているから。

---

## まとめ・今日からできること

- **ネタ収集・構成・執筆の3つを自動化**すると副業が週5時間で回る
- 仕組みを作る初期投資は1〜2週間かかるが、その後は毎週回収できる
- 「時間がない」の大半は、仕組みで解決できる

**今日からできること：CLAUDE.mdに自分の文体ルールを書く**

記事のスタイル（文体・禁止事項・ターゲット読者）をCLAUDE.mdに書くだけで、Claude Codeの出力品質が上がる。30分でできる。まずここから始めてください。
