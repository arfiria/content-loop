# Claude Code Routinesを設定したら副業の記事生成が寝ている間に終わるようになった

## はじめに

副業のために毎晩2時間作業していた。Routinesを設定した翌日から、朝起きたら記事の下書きができていた。

「副業をやりたいけど時間がない」――これは会社員が副業を諦める最大の理由だ。平日は残業、週末は体力が残らない。意志力だけでは続かない。

この記事では、Claude Code Routinesを副業に活用して「能動的な作業時間をほぼゼロ」にした方法を説明する。

自分は通信業界の会社員をしながら、Claude Codeを使った副業フリーランスをしている。Routinesを導入してから、副業の作業時間が週15時間から約2時間まで圧縮できた。何を設定したか、すべて公開する。

---

## Claude Code Routinesとは何か

2026年4月にリリースされた機能で、「Claude Codeにやることを登録しておくと、設定したタイミングで自動的にクラウド上で実行してくれる」仕組みだ。

最大のポイントは「PCをつけていなくても動く」という点だ。Anthropicのクラウドインフラ上で処理されるため、会社で仕事をしている昼間も、眠っている夜中も、Claudeは指示通りに動き続ける。

### Routinesのトリガー3種類

| トリガー | 使い方 |
|---------|--------|
| スケジュール | 「毎朝5時」「毎週日曜21時」など時刻を指定 |
| GitHubイベント | PRのマージ・issue作成をきっかけに起動 |
| API | 外部サービスから呼び出す |

副業のコンテンツ自動化には「スケジュール」トリガーだけで十分だ。

### 1日に使える回数（2026年5月時点）

- **Proプラン**: 1日5回まで
- **Maxプラン**: 1日15回まで
- **Team/Enterprise**: 1日25回まで

Proプランの5回で、副業の自動化はしっかり回せる。自分はProプランで毎日3〜4回を使っている。

---

## 副業で設定した3つのRoutines

### Routine 1: 毎朝5時 ─ ネタ収集

```
起動時刻: 毎日 05:00
作業内容: WebSearchでAI副業の最新情報を5件収集し、
         content-atoms.csvに追記する
所要時間: 約10分（クラウド上で処理）
```

朝起きると、Claudeが夜中に集めた今日のネタが5件並んでいる。自分がやることは「どのネタで行くか」を選ぶだけ。収集作業は完全に不要になった。

### Routine 2: 毎朝5時30分 ─ 企画案の自動生成

```
起動時刻: 毎日 05:30
作業内容: 収集されたネタをもとにNote記事・X投稿の企画案を自動生成し、
         proposals/YYYY-MM-DD.mdに保存して停止
所要時間: 約15分
```

朝7時に通知を確認して、proposals ファイルを開く。気に入った案に「approved」と書き換えるだけ。ここにかかる時間は5分だ。

Claudeは指示通りに止まって待っている。承認は人間がやる。この設計が重要だ。

### Routine 3: 毎週日曜21時 ─ SEOレビュー

```
起動時刻: 毎週日曜 21:00
作業内容: 過去7日間の公開記事をSEOスコアリングし、
         修正優先順位をつけたレポートを自動生成
所要時間: 約30分
```

月曜に「先週の記事のどれを直すべきか」がレポートで届く。自分は月曜の夜に30分でそのレポートを元に修正を入れる。SEO改善の判断を自分でしなくて済む。

---

## 実際の1週間のスケジュール（Routines導入後）

| 時間 | 自分がやること |
|-----|-------------|
| 毎朝7:00 | 企画案を確認→承認（5分） |
| 毎晩21:00 | 生成された記事をNoteにコピペ投稿（10分） |
| 毎週月曜夜 | SEOレポートをもとに記事修正（30分） |

**週の合計作業時間: 約2時間**

Routinesを導入する前は、ネタ収集・企画・執筆・投稿まで毎日2〜3時間かけていた。それが今は、1日15分+週次30分になった。

副業に「意志力」を使わなくて済むようになった。これが一番大きな変化だ。

## まとめ・今日すぐ設定できること

- Claude Code Routinesは2026年4月リリース。Proプランで1日5回使える
- スケジュールトリガーを使えば、寝ている間にコンテンツ生成が動く
- 承認だけ人間が行う設計にすることで、品質を保ちながら自動化できる

Claude Codeの環境がまだ整っていない場合は、[Claude Codeはターミナルで使うな。Cursorで使え。](https://note.com/umioto_lab/n/n98a7d4960e69)を先に読んでほしい。

**まず今日やること:**

公式ドキュメント（code.claude.com/docs/ja/routines）を開いてRoutineを1つ作成する。最初は「毎朝5時にニュースを3件収集してファイルに書く」だけの指示でいい。それだけでいい。

1回のRoutineが動けば、「自分がいなくても仕事が進む」感覚がつかめる。そこから設計を広げていく。

次回は、Routinesで使っている実際のプロンプト設計を公開する予定だ。
